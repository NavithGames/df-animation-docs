Notes (AI vs player-ride, rigging sockets, caveats)

  • AI-operated vs player-ride: every asset here is an AI behaving (gunner aims/fires, pilot steers, RCV drives) — AI possesses a pawn + runs a BT. The player enter/exit-ride system is a SEPARATE gameplay system, not these BTs. BTTask_Drama_UseMachineGun_Vehicle touches the seat layer via DFMAIEnvironmentInteractComponent + BP_SceneMachineGun_ForRaid_Vehicle — that's the boundary where AI plugs into the shared mounted-weapon/seat actor the ride system also uses.
  • Rigging-socket connection: turret yaw/pitch + gunner seat are properties of the vehicle/MG actor's skeletal RIG (sockets/bones), not the BT. The BT only computes a focus direction; the MountedWeaponComponent converts it to clamped yaw/pitch on the turret socket and places the gunner at the seat socket. So the BT is reusable but depends on the vehicle rig exposing turret-yaw / turret-pitch / gunner-seat sockets — wire those in (ties to the vehicle rigging discussed with ani49).
  • Engine-default (reuse): Selector/Sequence/SimpleParallel/RandomSelect, Blackboard/Cooldown/ForceSuccess/Loop/TimeLimit decorators, Wait/RunBehavior, BTService/BTTask BlueprintBase.
  • DF C++ (reimplement): DFMAIHeavyMachineGunComponent, DFMAIEnvironmentInteractComponent, DFMAICharacterMovementComponent, MultiWeaponManagerComponent, structs RepSteeringData/VehicleBoss04SkillState, AIHelicopterPatrolJob, GP* base (GPGameMode/GPCharacter).
  • Caveat: exact fire rates, traverse limits, missile homing constants, steering gains are NOT in json (cooked). Import the curve assets (RCVDelayHomingWithDist, RCVAITurnCurve, SuicideBlinkCurve, HelicotperFireZHight) for the tunable reference values.
