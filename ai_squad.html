DF Squad / Group AI - Formation + Shared Threat

Rebuild blueprint for WR2 (UE5.7). DF behavior trees are cooked + depend on DF C++ nodes, so they are not importable — these are reconstruction guides. · prepared by Wjna · 2026-06-24

IndexPVE OverviewPVE DetailPvPSquad
Squad AI is a thin BT layer (sprint + advance) plus a C++ group/formation system (AIGroupPatrolJob, FormationName) and a decentralized shared-threat bus (DFMAIThreatRecordComponent). Each squad member still runs its own per-soldier cover-combat underneath — see PVE Detail.

Asset inventory

  • Squad/: BT/Fight/BT_SquadFight_Sprint, BT/Fight/BT_Child_8DirMove_Shoot, BT/BTDecorator_CheckBehindTheBunker, BT/BTDecorator_CheckGroupAnim, BT/BTService_LevealSetAnimTime, BT/BTTask_SetBestFightDist, Common/FormationName (enum, 10 presets: shield+4, shield+RPG+3, wedge, circle, sentry, ...).
  • GroupAI/: mostly scripted-scene/interaction goals + Temp scaffolding (Enum_EachGoal, BPScenesAIStateComponent, BTD_Base/BTT_Base) — NOT the combat-squad system.
  • Real squad-combat formation logic lives in C++ AIGroupPatrolJob + BP_Drama_Group_Patrol (AI_SOL_V2/Drama_SOL, Boss/*_Group_Benchmark), consuming FormationName — outside the two named folders.
  • Threat glue (Common/Task): BTTask_BoardcastThreat, BTTask_PrefectureBoardcastThreat.

Main flow

Group layer assigns each member a formation slot / advance point (G_EQSLocation); member sprints there, then runs individual cover-combat; threats are shared by broadcast.

Notes (layering, node types, rebuild order)

  • Engine-default (reuse): Sequence, RandomSelect, SimpleParallel, ForceSuccess, RunBehavior.
  • DF C++ (reimplement): DFMAITask_AdvancedMoveTo, DFMAIService_WithMotionPlanFilter, DFMAIThreatRecordComponent (the squad glue), AIGroupPatrolJob (formation slot manager), DFMAIController, enums EAIThreatFlagName/EAIThreatDetailName/EAIThreatEvent.
  • _C Blueprint (recreate): BTTask_MoveState_C, BTTask_Speak_C, BTTask_SetBestFightDist, BTTask_BoardcastThreat, BTTask_PrefectureBoardcastThreat, BTDecorator_CheckBehindTheBunker, BTDecorator_CheckGroupAnim.
  • How it layers: the group layer (AIGroupPatrolJob + FormationName + BP_Drama_Group_Patrol) computes each member's slot/advance point and writes G_EQSLocation; BT_SquadFight_Sprint sprints each member there with a bark; then each member drops into its OWN cover-combat (SetBestFightDist -> CheckBehindTheBunker -> BT_Child_8DirMove_Shoot). Shared awareness is decentralized: every member broadcasts sensed threats via DFMAIThreatRecordComponent, so the squad converges on the same target while each soldier still runs individual cover-combat.
  • Rebuild order: (1) ThreatRecordComponent bus + broadcast task, (2) group manager owning members + assigning FormationName slot offsets -> write move points, (3) squad sprint/advance BT, (4) keep the per-soldier cover-combat (see PVE Detail) unchanged underneath.
  • Caveat: BT_SquadFight_Sprint as exported is minimal (sprint+bark+move); the real formation-keeping/suppression/role intelligence is C++ (AIGroupPatrolJob) + Drama group BPs, whose runtime logic is cooked.
