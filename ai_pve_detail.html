Subtree-level detail of the PVE infantry cover-combat. This is the "linked subtrees + customizable behavior per situation" layer: a cover EQS pipeline feeds a tagged cover stance, then a difficulty-weighted in-cover loop cycles idle / peek / blind-fire / reposition, dispatching to weapon-class shoot subtrees. See PVE Overview for the top-level loop.

Notes (node types, blackboard, caveats)

  • Engine-default (exist in 5.7, reuse): Selector, Sequence, SimpleParallel, Wait, RunBehavior, ForceSuccess, Loop.
  • DF C++ (no code in WR2, MUST reimplement): GPEQSGenerator_RoundArea, GPEnvQueryTest_PathTest, EnvQueryTest_BasicCoverCheck/BasicVisibilityCheck/SurroundingVisibilityCheck/AvoidOtherBot/DistToZone/AvoidGrenade/NavRayCheck, DFMAITask_AdvancedMoveTo/MoveToCover/RequestToCoverState/Shoot/PlaySpecialActing_Name, DFMAIService_WithMotionPlanFilter, GPBTTask_RunBlackboardBehavior, BTComposite_RandomSelect, RandomTimeLimit, plus runtime systems DFMCoverContext / DFMAICoverManager / DFMAIThreatRecordComponent / DFMAIDiffcultyComponent.
  • _C Blueprint nodes (logic cooked = only defaults extracted, behavior inferred from names): RunCoverEQS_C, MoveToCover_C, RequestCoverStateSimple_C, ActiveNewAICoverContext_C, CalcCoverBlindShootLoc_C, CopyCoverLocToBB_C, SetCurTimeToBB_C, SetBoolValue_C, JudgeCoverType_C, JudgeCoverBehaviorProbability_C, CheckCoverState_C, CoverCheckCanShootTarget_C, Random_C, SetFocus_C, IgnoreHistoryLoc_C.
  • Blackboard BB_AI (AI2/DecisionMaking/BehaviorTree/BB_AI). Keys: G_EQSLocation, G_MainTarget, G_MainTargetChanged, G_MainTargetDistance, G_HasAimedTargetKey, CoverLocation, CoverDirection, CoverTypeKey, CoverEqsFailCountKey, InterruptKey.
  • In-cover behavior set is difficulty-weighted: FullExpose / LyingBlindFire / BlindFire / Peek / LeanFire via DFMAIDiffcultyComponent — AI aggression scales with difficulty. Replicate as a difficulty data asset feeding the weighted-random decorators. (This is the 'customizable behavior per situation / linked subtrees' layer.)
  • Values read directly: RandomSelect [15,15,70]; BlindShoot offsets 300,600,0 / 300,-600,0; RunCoverEQS radius 1000 / maxpath 3000. Some EQS names overstate radius (MoveBack_800 actually R=700, TrimPos_600 R=700). Composite/decorator order = json child array order (top->bottom = eval order).
