Boss AI = a shared cover-combat base (RunCoverEQS / MoveToCover / DFMAITask_Shoot on BB_SOLAI_V2_Template) wrapped by a per-boss L3 root that swaps Patrol/Leisure vs Fight, where Fight is a Selector of skill subtrees gated by distance-band + cooldown. Bosses with no own BT ride the base directly.

Notes (layering, group-benchmark, node types, caveats)

  • Layering: every boss = common cover-combat base + per-boss skill set + phase/distance logic. Base lives in Common/Task (RunCoverEQS, MoveToCover, RequestCoverStateSimple, BaseFightTarget) + Common/EQS/Cover + BP_CoverManager on BB_SOLAI_V2_Template. Bosses with NO own BT (Firebat, HeavyMachineGun, RPG, Shielder) ride this base directly — only the character BP + curves/abilities differ. Bosses with a Fight/ folder (Maurice, Reis, Boss03) add a Selector of skill subtrees on top. (See PVE Detail for the shared base.)
  • Distance-band selection is encoded in subtree filenames (Maurice EarlyHand/LateHand_Mid|Far, Melee_Near, Grenade_Near|Far; Reis CloseToBoss_800/1000/1200) + BTDecorator_DistanceToTarget_C. Cooldowns via engine Cooldown (10.0s seen) + RandomCooldown/Random_C to avoid same-skill spam.
  • Group-benchmark pattern: BP_Drama_BossNN_Group_Benchmark + BT_BossNN_GroupMove_Benchmark (Boss05/06) are squad-formation reference trees a commander runs to drive minions (BP_Boss05SquadManager, Boss05_CallAround, GetSquadCommanderBBVector) — the group-coordination layer (see Squad page); minions still run the common cover base.
  • Engine-default (reuse): Selector/Sequence/SimpleParallel/RandomSelect, RunBehavior/Wait, Cooldown/Blackboard/Loop/ForceSuccess.
  • DF C++ (reimplement): DFMAITask_Shoot, DynamicWait, DFMAITask_Boss03Skill*, EQS generators/contexts. _C Blueprint (rebuild, cooked): InitMauriceAction_C, DistanceToTarget_C, CheckCanShootTarget_C, CheckHealth_C, CommonLineTrace_C, GrenadeDetectedSpecifyRadius_C, RunCoverEQS_C.
  • Caveat: numeric thresholds (distance bands, cooldown durations, HP phase cutoffs) live in the cooked Blueprint/CDO — BTDecorator_DistanceToTarget_C Properties came back empty. Graph TOPOLOGY is fully recoverable; tuning values must be re-derived by play-testing.
