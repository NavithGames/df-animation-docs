7. PasOut System — Drive Shooting Detail

PasOut is Delta Force's vehicle "lean-out shooting" system. A passenger hangs out of the vehicle and maintains a full 3D aim space while the vehicle is moving. The naming convention encodes all the information needed to build a BlendSpace or AimOffset.

Naming Convention

Full pattern:

{Vehicle} _ Vehicle identifier — ATV, ORV, MotorBoat, etc.
Player _ Always "Player" — distinguishes from vehicle-self anims.
{Perspective} _ 1P or 3P. PasOut is always 3P (you never see yourself leaning out in first person).
PasOut{Seat} _ Seat position — B (Back), BL (Back-Left), BR (Back-Right), FR (Front-Right).
{Direction}{Elevation} Aim direction + optional elevation suffix.

Direction Tokens

TokenMeaningRelative To
FForwardVehicle forward direction
BBackwardVehicle rear
LLeftVehicle left side
RRightVehicle right side
FLForward-Left diagonal45° front-left
FRForward-Right diagonal45° front-right
BLBackward-Left diagonal45° rear-left
BRBackward-Right diagonal45° rear-right

Elevation Suffix

SuffixMeaning
(none)Level — horizontal aim
upElevated aim — shooting upward (anti-air, hilltop)
downDepressed aim — shooting downward (valley targets)

Full Example Breakdown

ATV_Player_3P_PasOutB_Fdown │ │ │ │ │ └── Elevation: looking DOWN │ │ │ │ └──── Direction: Forward │ │ │ └──────────── Seat: B (Back passenger) │ │ └─────────────── Perspective: 3P │ └────────────────────── Always "Player" └────────────────────────── Vehicle: ATV

BlendSpace Usage

These animations form a complete 2D AimOffset grid:

Fup │ BLup ── F ── FRup │ │ │ BL ─────CENTER──── FR ← horizontal aim (level) │ │ │ BLdn ── B ── BRdn │ Bdown

In UE5, set up a BlendSpace2D with Yaw on the X axis (−180 to +180) and Pitch on the Y axis (−45 to +45). Each PasOut animation maps to a grid point.

8. Import Notes for Package2

StepDetail
Source Files ExportedVehicles/DeltaForce/Content/Models/Vehicles/ — PSA files organized by vehicle subfolder.
Conversion Use DFPakExplorer export-asset-ueformat to convert .psa → .ueanim. No FBX intermediate needed.
UE5 Import UEFormat plugin handles .ueanim directly. Bone auto-matching selects the correct skeleton on import.
Player Anim Skeletons 1P anims → C_Test_001_1P_Skeleton (83 bones, Mixamo names). 3P anims → C_Test_001_3P_Skeleton (113 bones, Mixamo names).
Vehicle-Self Skeletons Each vehicle type has its own skeleton generated when the vehicle PSK was imported. Weapon-self animations reference the vehicle mesh skeleton, not the character skeleton.
Skeleton Assignment Import with skeleton=None first (auto-generate), then reassign via UE5 Python after import. Never share skeletons across vehicle types during batch import — the FBX importer can corrupt shared skeletons.
Naming in UE5 Keep the original PSA filename as the AnimSequence asset name — the naming convention is load-bearing (PasOut direction tokens must be readable by the AnimBP).
Pipeline verified: The UEFormat direct import pipeline (PSA → .ueanim → UE5) was confirmed working for vehicle animations. 158/158 test assets imported at 100% success rate during the UEFormat validation run (2026-02-26).
Batch import limit: Do not import all 1,327 animations in a single batch via MCP — UE5 will crash. Split into phases by vehicle type or category. Recommended batch size: ≤150 animations per run. 
# Convert PSA to ueanim (run from DFPakExplorer directory)
DFPakExplorer.exe export-asset-ueformat --path Models/Vehicles/ATV/ --output ueanim/ATV/

# UE5 Python import (run in UE5 Python console)
exec(open(r"F:\UE_DF_Package2\Scripts\ue5_import_vehicle_anims.py", encoding="utf-8").read())
