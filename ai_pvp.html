PvP-mode AI is an objective/event-driven "bot-as-player" system: one shared BehaviorTree + rich Blackboard (BB_PvPAI) steered by C++ threat-curve scoring (target priority), a Director/AIJob layer (capture points, battle routes, rescue), and EQS pathing-grid queries for advance/cover — distinct from PVE's scripted spawn encounters.

  • BehaviorTrees: PvP folder ships reusable BT node BPs (Tasks BTTQ_FindTargetActor, BTTQ_Shoot, BTTQ_SetFocus, BTTQ_SkillFire, BTTQ_ActivateSkillBySlot, BTTQ_SwitchToMainWeapon, BTTQ_Slide, BTTQ_FastFindPoint; Services BTSQ_ChangeMoveState, BTSQ_AccuIdleTime; Decorators BTDQ_IsArmedForce, BTDQ_BBVectorIsSet) + 3 real BehaviorTree test assets (BT_Breakthrough_AnniversaryEgg_*). The live PvP combat tree references BB_PvPAI but its .uasset is not in this folder (cooked-out).
  • Blackboard: BB_PvPAI (70+ keys).
  • EQS (4): EQS_PvPAdvance, EQS_HangAroundSelf_1500, EQS_Guard_Egg_500, EQSTestPawn.
  • Threat/Route curves: Threat (Vehicle/Sight/Hearing/AirHit/TeamHurt), BattleRouteThreat (2Cost, DecayWithDist/Time, GroundVehicle/NormalKill/SceneWeapon), HitRate (LMG/SMG), Vulnerability (CQB/Gunner/Medic/Sniper/Shotgun +Mobile), Rescue (SafetyScore*), Director (BattleLineDistScore).
  • CoreLogic: BP_AIPlayerController (casts DFMAIPlayerController; reads Team/Camp/ArmedForceID), BP_PVPAIComponent (/Script/BattleField PvpAIComponent; CalcStickySightThreat, CalcStickyVul, BPCalcVehicleThreatValue). Subsystems: BP_TDMAISubsystem, BP_CampGuardPoint. Navigation: BP_JumpParachuteNavLinker (spline nav-link: parachute/zipline traversal).

Threat-curve & navigation

  • Target selection is curve-scored, not nearest-first. CalcStickySightThreat / CalcStickyVul / BPCalcVehicleThreatValue feed distance into CurveFloats: VehicleThreatCurve 100@0m -> 0@100m; Sight vs SightNotInSight separate visible vs last-known; Hearing adds audio; TeamHurtThreatDecayWithT raises threat of whoever hurt a teammate then decays it. 'Sticky' = hysteresis so the bot doesn't ping-pong targets.
  • Per-class Vulnerability curves (Sniper/Gunner/Medic/CQB/Shotgun +Mobile) make a bot's own exposure role-dependent — drives push vs hold aggression.
  • Route/navigation is threat-cost driven: BattleRouteThreat2Cost converts accumulated route threat into pathing cost; DecayWithDist/Time decay it; NormalKill/SceneWeapon/GroundVehicle curves stamp threat onto routes from recent kills/weapons/vehicles. Bot picks ATargetBattleRoutePoint by lowest threat-cost -> flows along the safest route toward the objective/battle line (Director/BattleLineDistScore, RelayPointsDistanceWeight).
  • Tactical micro-positioning = EQS_PvPAdvance (pathing-grid, end near cover, not behind enemy/self, short path to enemy last loc). Special traversal (deploy parachute / zipline) via BB bShouldUseZipline + BP_JumpParachuteNavLinker spline.

Notes (PVE vs PvP, node types, caveats)

  • PVE vs PvP: PVE = scripted-encounter / spawn-driven extraction. PvP = mode/objective-driven (TDM via BP_TDMAISubsystem, Breakthrough, capture-point/flag jobs), team+faction aware, simulates a human player (locomotion states, peek/push, slide, ability slots, per-class vulnerability), selects targets+routes by continuous threat-curve scoring. A Director coordinates the squad across the match.
  • Engine-default (reuse): Selector, Sequence, Wait, Loop, ForceSuccess, EnvQueryGenerator_PathingGrid, EnvQueryTest_Dot/Distance.
  • DF C++ (reimplement): DFMAI* (DFMAIDecorator_IsDead, DFMAIPlayerController), GP* (GPEQSGenerator_RoundArea, GPEnvQueryTest_PathTest, AIJob), PvpAIComponent + threat math, EnvQueryTest_NearCover.
  • _C Blueprint (rebuildable): all BTTQ_/BTSQ_/BTDQ_ nodes, BP_AIPlayerController/BP_PVPAIComponent wrappers, BP_CampGuardPoint, BP_TDMAISubsystem.
  • Caveat: only 3 anniversary-event test trees are real BehaviorTree objects; the production PvP tree .uasset is not in the folder. .json exposes structure/properties/BB keys/curve values/refs, NOT the scoring formulas inside DFMAI*/PvpAIComponent C++ or the _C event graphs. Rebuild from curve shapes + key names.
