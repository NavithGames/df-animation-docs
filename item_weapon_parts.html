1. 발견 요약

Delta Force의 건스미스(Gunsmith) 시스템은 전용 DataTable 기반이며, 4개 레이어로 구성된다.

무기(Rec/RecFunction) → 슬롯(PartSocketsTable) → 파츠(PartsDataTable) → 기능(PartsFunctionDataTable)
  • 파츠는 EAdapterItemType enum으로 슬롯 타입이 구분됨
  • 각 파츠에는 FunctionId가 있어 PartsFunctionDataTable과 연결, 스탯 보정을 정의
  • 메시 부착은 WeaponPartsMeshSocketNameInfo에 소켓 위치/회전 하드코딩
  • 파인 튜닝(FTune)은 TunePartFunctionTable의 슬라이더로 추가 보정 가능
  • 호환성 규칙은 PartsRuleTable + PartsRuleGroupTable + PartsForbidRuleTable 3중 구조

14. 건스미스 시스템 플로우 (종합)

1. 무기 선택
   └─ RecFunctionTable[RecId]
       ├─ DefaultPreset → GunPresetTable → 기본 파츠 구성
       └─ BPClass = BP_WeaponModular (모든 무기 공통)

16. 불확실 항목

  1. 파츠 ItemId와 실제 파츠 타입(슬롯) 매핑: PartsDataTable에 슬롯 타입 필드 없음. PartsRuleTable의 SocketId로 역추적해야 함 — 직접 조회 필요.
  1. FunctionId 중 StaticAttributeModify Param1 전체 목록: 추출된 샘플에서 ChangeClipMode, ChangeClipTime만 확인. 반동(Recoil)/ADS속도/명중/이동속도 등 속성 키는 추가 로우 분석 필요.
  1. Lua 건스미스 UI 로직: LuaSource는 컴파일된 LuaC 바이너리 — 심볼(함수명/상수)은 읽히나 실제 로직 상세는 역컴파일 없이는 확인 불가.
  1. 파츠 메시 대부분 미추출: Models/Weapons/Parts/ 아래 JSON export가 극히 일부(10개 미만). 실제 메시 경로는 WeaponPartsMeshSocketNameInfo CSV에서 /Game/Models/Weapons/Parts/FroS/.../ 패턴으로 다수 확인되나 pskx/uasset은 별도 추출 필요.
  1. EAdapterItemType 전체 enum 값: 확인된 값 (PistolGrip, GunBarral, Stock, Handguard, Magazine, Muzzle, ForSight, Foregrip, Sight, Bipod, TacticalDevices, Receiver). 활 관련 소켓(Lua 심볼)으로 보아 근접/특수 무기 전용 타입 추가 존재 추정.
  1. PartsMergeableDataTable의 AppearanceId: 대부분 0 — 스킨 관련 외관 ID로 추정되나 연결 테이블 미확인.
